# Lesson 2: Event Viewer Deep Dive

# Learning Goal
* Open Event Viewer.
* Understand every section.
* Know where Windows stores different types of events.
* Decide which log to check based on an alert.
* Start thinking like an investigator instead of memorising Event IDs.

Event Viewer is the built-in Windows tool for viewing Windows event logs. It organises logs into categories such as Application, Security, and System, making it the primary place to investigate activity on a Windows machine.

# What is Event Viewer?

* Think of Event Viewer as a file explorer for logs.

* Just like File Explorer organises files into folders, Event Viewer organises events into different log categories.

Windows Computer
       │
       ▼
Windows Event Logs (.evtx)
       │
       ▼
Event Viewer
       │
       ▼
SOC Analyst

* Event Viewer does not create logs.

* It only displays logs that Windows has already recorded.

# Event Viewer Structure

* When you open Event Viewer, you'll see something like:

Event Viewer

We'll focus on Windows Logs first because these are the logs you'll use most often in SOC investigations.

# The Five Windows Logs
1. Application

Stores events generated by applications.

* Examples:

- Google Chrome crashes
- SQL Server errors
- Microsoft Office issues
- Adobe Reader problems

Question:

Is the operating system broken?

No.

Only the application has a problem.

2. Security ⭐ (Most Important)

This is the log SOC analysts check most often.

* Examples:

- User logon
- Failed logon
- Password change
- New user creation
- Account lockout
- Privilege changes

Most security investigations start here.

3. System

Generated by Windows itself.

* Examples:

- Driver loaded
- Windows startup
- Windows shutdown
- Service started
- Disk errors
- Hardware issues

Question:

Why did Windows suddenly restart?

Start with the System log.

4. Setup

Contains installation-related events.

* Examples:

- Windows Updates
- Feature installations
- OS upgrades
  
5. Forwarded Events

Used in enterprise environments.

Imagine:

PC 1
PC 2
PC 3
PC 500
      │
      ▼
Central Windows Server
      │
Forwarded Events

FACT: Organisations can forward logs from many computers to one location.

# Which Log Should You Check?
Scenario	Log
Failed login	Security
User created	Security
Windows restarted unexpectedly	System
Printer driver failed	System
Chrome crashed	Application
Windows Update failed	Setup
Event Levels

# Every event has a severity level.

# Information

Everything is normal.

Example:

User logged in successfully.
Warning

# Something may become a problem.

Example:

Disk space is getting low.
Error

# Something failed.

Example:

Service failed to start.
Critical

# A serious issue.

Example:

Windows crashed unexpectedly.

Remember:

FACT: Error does not always mean "cyberattack."

One of the biggest beginner mistakes is assuming every warning or error indicates malicious activity. Many are part of normal system operation.

# Investigation Mindset

- Suppose you receive this alert:

- Multiple Failed Logins

- Instead of immediately assuming an attack, ask:

- Which log should I check?
- When did it happen?
- Which account?
- Which computer?
- Is this normal for this user?
- Did any login eventually succeed?

That's how analysts investigate.
